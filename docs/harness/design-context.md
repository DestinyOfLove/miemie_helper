# Design Context

## Users

- 主要用户是政府场景下的日常办公使用者。
- 使用场景是本地处理文档搜索、归档和辅助性业务操作，目标是减少重复劳动并提升处理效率。
- 当前已知核心诉求不是“展示型产品体验”，而是“容易上手、稳定可靠、节省时间”。
- 当前前端按桌面端优先考虑，不做手机端体验优化。
- 使用平台需要同时考虑 macOS 和 Windows 桌面环境。

## Brand Personality

- 关键词：现代、专业、高效
- 界面应传达克制、可信、清晰，而不是活泼、炫技或消费级娱乐感。
- 用户完成任务时应感受到：
  - 信息容易找到
  - 操作路径短
  - 关键状态明确
  - 不需要学习成本就能开始工作
  - 视觉上不是陈旧后台，而是现代桌面效率工具

## Aesthetic Direction

- 双主题：同时支持 light 和 dark，但都应保持办公软件语义下的稳定感与高可读性。
- 视觉方向优先考虑“现代专业办公工具”而不是“营销页面”。
- 实现策略优先采用成熟开源 UI 库，而不是自建大量基础控件。
- 现代感的来源应是：
  - 更清晰的层级与留白
  - 更精炼的色彩系统
  - 更轻的边框与更干净的表面处理
  - 更像现代效率软件的导航、列表和状态设计
- 建议使用低饱和中性色作为大面积背景，辅以克制的蓝 / 青蓝色系统色表达主操作、选中和信息状态。
- 避免：
  - 过强装饰性渐变
  - 夸张动效
  - 过度卡片化
  - 玩具化插画和表情化反馈
  - 明显 AI 模板感的炫彩深色界面
  - 老式政务系统式的厚重蓝条、强描边和高密度挤压布局

## Theme Guidance

- Light theme
  - 以浅灰白背景、深灰文本为主，重点信息使用稳重蓝色强调，避免纯白底配生硬纯蓝。
- Dark theme
  - 不是纯黑；使用带冷灰或蓝灰倾向的深色背景，保证长时间阅读舒适度。
  - 暗色模式要像专业生产力工具，而不是霓虹科技风。
- Color direction
  - 推荐主色方向：蓝、青蓝、石墨灰。
  - 避免高饱和紫、荧光色、大面积纯红纯绿。

## Accessibility Baseline

- 当前不以正式 WCAG 达标为项目目标，但仍应保持基础桌面可用性：
  - 正文与操作文本要有足够对比度
  - 交互状态要有明确 hover / focus / active 区分
  - 动效保持克制，并避免把信息只编码在颜色里

## Design Principles

1. 优先让用户更快完成任务，而不是先让界面显得“特别”。
2. 重要信息必须可快速扫读，表格、状态、按钮层级要一眼分明。
3. 一切装饰都要服务于可信度和现代感，不能制造额外认知负担。
4. 默认采用专业办公产品语气，避免过度热情、拟人化或情绪化文案。
5. 在 light / dark 两套主题下保持同一套信息层级和交互逻辑。
6. 现代化不靠炫技，而靠秩序、节奏、细节和稳定的交互反馈。
7. 优先复用成熟库能力，避免自建大量基础轮子增加维护负担。

## Reference Directions

这些是可借鉴的方向，不是要求复制：

- GOV.UK Design System
  - 借鉴点：信息层级直接、表单与状态表达清晰、几乎没有装饰噪音。
- Apple macOS System Settings / Finder 列表体验
  - 借鉴点：适合桌面场景的清晰分组、克制留白、稳定的 light / dark 切换。
- Linear
  - 借鉴点：高信息密度下仍保持秩序、交互反馈干净利落、整体气质现代。
- Notion 数据库视图
  - 借鉴点：任务型界面对“筛选、查找、列表浏览”的高效组织方式。

## Anti-Reference Direction

- 不要做成营销官网风格
- 不要做成夸张炫技的深色玻璃拟态
- 不要做成色彩花哨、强调情绪表达而弱化任务效率的消费级 app
- 不要做成传统旧式后台那种厚导航条、重阴影、重边框、满屏蓝色按钮的风格
