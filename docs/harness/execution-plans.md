# Execution Plans

## When To Write A Plan

以下任务先写 `docs/plans/<YYYY-MM-DD>-<topic>.md` 再实现：

- 修改搜索、索引、提取、归档等跨模块流程
- 修改前后端接口契约、返回字段或页面主流程
- 修改安装、启动、数据目录、文件系统副作用
- 做迁移、兼容性处理、批量重构
- 调整 harness 结构、仓库级规则或验证流程

小范围单文件修复、纯文案调整、局部样式修改通常不需要计划文件。

## Minimum Plan Template

计划至少包含这些内容：

- 目标
- 影响面
- 关键文件 / 模块
- 风险与回滚点
- 验证命令
- 完成标准

## Working Rules

- 计划文件是执行记录，不是随手草稿。
- 如果范围变化，更新原计划，不要新开多个互相冲突的版本。
- 完成后保留计划，便于后续 agent 追溯上下文。
